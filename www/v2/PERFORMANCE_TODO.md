# План оптимизации производительности

**На основе аудита от 8 марта 2026**  
**Приоритеты:** P0 (критично) → P1 (высокий) → P2 (средний)

---

## 🔴 P0 — Критические (исправить немедленно)

### 1. Утечка памяти в OptimizedVideo.tsx
**Файл:** `src/components/OptimizedVideo.tsx`  
**Проблема:** HLS инстансы не освобождаются при быстром переключении  
**Решение:**
- [ ] Добавить cleanup в useEffect только при размонтировании
- [ ] Освобождать object URLs при смене источника
- [ ] Добавить флаг isMounted для предотвращения утечек

**Ожидаемый эффект:** -30% памяти при долгой сессии

---

### 2. Утечка Object URLs в mediaPreloader
**Файл:** `src/services/mediaPreloader.ts`  
**Проблема:** `URL.createObjectURL()` вызывается без последующего `revokeObjectURL()`  
**Решение:**
- [ ] Кэшировать object URL внутри MediaItem
- [ ] Добавить метод `cleanup()` для освобождения всех URL
- [ ] Вызывать cleanup при сбросе preloader

**Ожидаемый эффект:** -15% памяти

---

### 3. Race condition в recommendationService
**Файл:** `src/services/recommendationService.ts`  
**Проблема:** `Math.random()` делает сортировку нестабильной  
**Решение:**
- [ ] Вынести рандомизацию в отдельный шаг
- [ ] Использовать детерминированный seed на основе productId
- [ ] Добавить мемоизацию для getRecommendedProducts

**Ожидаемый эффект:** стабильные ре-рендеры

---

### 4. Отсутствие AbortController для fetch
**Файл:** `src/App.tsx`, `src/components/CheckoutSheet.tsx`  
**Проблема:** Запросы не отменяются при размонтировании  
**Решение:**
- [ ] Создать AbortController в useEffect
- [ ] Отменять запросы в cleanup функции
- [ ] Добавить обработку AbortError

**Ожидаемый эффект:** отсутствие утечек запросов

---

## 🟠 P1 — Высокий приоритет (исправить в течение недели)

### 5. Виртуализация в CheckoutSheet
**Файл:** `src/components/CheckoutSheet.tsx`  
**Проблема:** Рендеринг 1000+ DOM узлов при большом заказе  
**Решение:**
- [ ] Установить `@tanstack/react-virtual`
- [ ] Обернуть flatItems.map в виртуализированный список
- [ ] Добавить placeholder для элементов

**Ожидаемый эффект:** -20% памяти, -10% FCP

---

### 6. Мемоизация тяжёлых вычислений
**Файл:** `src/components/ProductDetail.tsx`, `src/components/CheckoutSheet.tsx`  
**Проблема:** Избыточные вычисления на каждый рендер  
**Решение:**
- [ ] Мемоизировать getGroupSummary с useMemo
- [ ] Мемоизировать flatItems с useMemo
- [ ] Мемоизировать calculateBaseTotal с useCallback

**Ожидаемый эффект:** -15% FCP

---

### 7. React.memo для компонентов
**Файл:** `src/components/CatalogSheet.tsx`, `src/components/MainPage.tsx`  
**Проблема:** Перерисовка при любом изменении состояния  
**Решение:**
- [ ] Обернуть CatalogSheet в memo с custom comparison
- [ ] Обернуть MainPage в memo
- [ ] Добавить сравнение props для избежания лишних рендеров

**Ожидаемый эффект:** -20% ре-рендеров

---

### 8. Debounce для polling и валидации
**Файл:** `src/App.tsx`, `src/components/CheckoutSheet.tsx`  
**Проблема:** Блокирующий main thread при валидации  
**Решение:**
- [ ] Добавить debounce 300ms для валидации формы
- [ ] Добавить экспоненциальную задержку при ошибках polling
- [ ] Использовать lodash.debounce или кастомную реализацию

**Ожидаемый эффект:** -10% нагрузки на CPU

---

## 🟡 P2 — Средний приоритет (исправить в течение месяца)

### 9. Оптимизация размера бандла
**Файл:** `vite.config.ts`  
**Проблема:** Bundle size ~450 KB  
**Решение:**
- [ ] Настроить tree-shaking для motion/react
- [ ] Добавить анализ бандла с `rollup-plugin-visualizer`
- [ ] Вынести тяжелые библиотеки в отдельные чанки

**Ожидаемый эффект:** bundle -55% (до 200 KB)

---

### 10. Стратегия кэширования stale-while-revalidate
**Файл:** `src/services/mediaPreloader.ts`  
**Проблема:** Cache API используется без стратегии обновления  
**Решение:**
- [ ] Реализовать стратегию: cache first, network update
- [ ] Добавить фоновое обновление кэша
- [ ] Добавить TTL для кэшированных ресурсов

**Ожидаемый эффект:** -40% FCP при повторном визите

---

### 11. Мониторинг Web Vitals
**Файл:** Отсутствует  
**Проблема:** Нет метрик производительности  
**Решение:**
- [ ] Установить `web-vitals`

- [ ] Настроить отправку метрик на сервер

**Ожидаемый эффект:** видимость проблем производительности

---

### 12. Поддержка современных форматов изображений
